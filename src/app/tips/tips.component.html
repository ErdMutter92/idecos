IDECOS: Release Notes

IDECOS V0.0.8 pre-alpha

  1. Added editor with tabbing and directory tree
  2. Added Smart detection of content type via file extension for:
    1. JavaScript
    2. TypeScript
    3. CSS
    4. Text
    5. Markdown
    6. HTML
    7. Python
    8. JSON
  3. Added temp change log page where future documentation module/plugin will be located.
  4. Added temp loading animation page where future settings/plugin manage page will be located.
  5. Added keybinding for saving documents (ctrl+s)
  6. Fixed issue that prevented the editor from saving.
  7. Fixed saved documents sometimes saves more then it should.
  8. Fixed srinking and stretching of editor outside of container when resizing window.
  9. The editor now remembers the last project opened and reopens it.
  10. Added context menu to editor directory to close project

Known Issues

  1. Application gets really slow when you have a large project
  2. Unable to open multiple directories/projects
  3. Editor does not load in directory files to reference in other documents.
  4. Unable to create new files/folders within a directory
  5. Tabs overflow not allowing access to those files when to many are open

Tips

  1. Open a file by clicking on it in the directory to the left
  2. Files indicate changes by switching to a solid white file icon, as well as an * in front of the open tab.
  3. Changes made to a file are preserved even when the tab is closed. You will need to save them before closing or you will lose your work.
  4. To save a document you must right click the tab (this is temporary and will not be the case after context menus are created)
